# UVM Hello World Example with Vivado Simulator

This repository contains a simple SystemVerilog UVM "Hello World" simulation project set up to run using the AMD/Xilinx Vivado Simulator (`xsim`).

## 📁 File Structure

The project consists of the following files:

*   **Source Files:**
    *   [uvm_hello_world.sv](uvm_hello_world.sv) - The top-level module ([top](uvm_hello_world.sv#L8)) containing the interface definition ([dut_if](uvm_hello_world.sv#L1)), dummy [dut](uvm_hello_world.sv#L4) instantiations, and the `initial` block that triggers the UVM test.
    *   [my_pkg.svh](my_pkg.svh) - Package container file that imports UVM package, includes UVM macros, and ties other source components together.
    *   [my_env.svh](my_env.svh) - UVM environment class ([my_env](my_env.svh#L1)) containing basic `build_phase` and `run_phase` methods.
    *   [my_test.svh](my_test.svh) - UVM test class ([my_test](my_test.svh#L1)) that creates the environment.

*   **Automation Scripts:**
    *   [run_sim.bat](run_sim.bat) - A batch script to compile, elaborate, and simulate the project in Vivado.
    *   [clean.bat](clean.bat) - A batch script to clean up temporary compilation and simulation outputs.

---

## 🛠️ Prerequisites & Setup

### Vivado Installation Path
The simulator scripts assume a default installation of **Vivado 2021.2** at `C:\Xilinx\Vivado\2021.2`. 
If you have a different version of Vivado installed, open [run_sim.bat](run_sim.bat) and modify the following line to match your path:

```batch
call C:\Xilinx\Vivado\<your_version>\settings64.bat
```

## 🚀 Running the Simulation

You can run the entire simulation workflow automatically or step-by-step from the command line.

### Option A: Automatic Script execution (Recommended)

1. Open PowerShell or Command Prompt.
2. Navigate to this folder.
3. Execute the simulation batch file:
   ```cmd
   .\run_sim.bat
   ```

### Option B: Step-by-Step CLI Execution

If you prefer to run commands manually, first open a Command Prompt, load the Vivado environment, and execute each stage:

1.  **Initialize Vivado Environment:**
    ```cmd
    call C:\Xilinx\Vivado\2021.2\settings64.bat
    ```

2.  **Compile Source Files (`xvlog`):**
    Compile the SystemVerilog package and the top-level testbench file. The `-L uvm` flag is required to compile against the pre-compiled UVM library bundled with Vivado.
    ```cmd
    xvlog -sv -L uvm my_pkg.svh uvm_hello_world.sv
    ```

3.  **Elaborate the Design (`xelab`):**
    Link the compiled modules, instantiate the design hierarchy, and build a simulation snapshot named `top_sim`.
    ```cmd
    xelab -L uvm top -s top_sim -debug typical
    ```

4.  **Run Simulation (`xsim`):**
    Execute the simulation. The `-R` switch runs the simulation immediately on startup (equivalent to running `run -all`).
    ```cmd
    xsim top_sim -R
    ```

## 🧹 Cleaning Simulation Outputs

Vivado simulation tools generate several intermediate compilation databases, log files, and journal files. To clear these and keep the workspace clean, run:

```cmd
.\clean.bat
```

This removes:
*   `xsim.dir/` (Simulation snapshot directory)
*   `*.log` (xvlog, xelab, xsim output logs)
*   `*.pb` (Protocol buffer files generated by Vivado compiler)
*   `*.jou` (Vivado journal logs)
*   `*.wdb` (Waveform database files)
